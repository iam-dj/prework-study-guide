HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The br element creates a line of empty space, or a line break.
  • The element (which stands for anchor) creates links to the same webpage or other webpages.
  • The p element represents a paragraph or block of text.
  • The img element contains information about images that are displayed on the webpage.
  • The head contains the metadata for our web app
  • The h1, h2, h3, h4, h5, and h6 elements represent the level of heading a given text block represents.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The asterisk selector means that we want to apply a rule to all the elements visible on the page.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • The period preceding card designates this as a class selector. Any HTML element assigned to the class, card, will gain these property assignments from the CSS
  • If two or more selectors will share the same styling they can be grouped together with a comma
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git status ~~ will show you what current branch you’re on
  • Stage your saved changes before you commit syntax: git add -A
  • Commit with a precise description of what you did - syntx: git commit -m "DESCRIPTION"
  • Pull source code from GitHub before you push your cahnges - syntax git pull origin main
  • Push you cahnges if you're up to date - syntax git push origin feature/NAME

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Conditionals are code structures used to test if an expression returns true or not.
  • An immutable value is one whose content cannot be changed without creating an entirely new value.
  • Strict equality tests to see if two values are equal. It returns a (Boolean) result. Syntax: ===
